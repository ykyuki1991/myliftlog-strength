# MyLiftLog Strength Planner

BIG3（ベンチプレス／スクワット／ハーフデッド／床引きデッド）強化向けの個人用トレーニング管理アプリです。
8日ローテーション × 4ローテで1ブロックの構成で、MAXを入力すると毎日の予定重量・回数・セットを自動計算します。
スマホ縦画面を最優先に設計した、ブラウザだけで動くPWA（オフライン対応）です。

## 特徴

- 完全静的（HTML/CSS/JS のみ） — GitHub Pages でそのまま公開可
- ログイン不要・バックエンド不要
- データはブラウザの `localStorage` に保存
- PWA対応（ホーム画面追加・オフライン起動）
- 外部ライブラリ未使用（依存ゼロ）
- 日本語UI／スマホ最適化
- レストタイマー内蔵（音 + バイブ + 画面表示）

## 使い方

### 起動
1. `index.html` をブラウザで開くだけで動作します（ローカルでは `file://` でも基本動作）。
2. PWAやService Workerまで含めてフル動作させたい場合は、簡易HTTPサーバ経由で開いてください。

```bash
# Python 3 がある場合
cd myliftlog-strength
python3 -m http.server 8080
# → http://localhost:8080 をブラウザで開く

# Node がある場合
npx serve .
```

### MAX設定
画面下「設定」タブから以下を入力します（初期値は要件どおり）。
- ベンチプレスMAX：115kg
- スクワットMAX：160kg
- ハーフデッドMAX：190kg
- 床引きデッドMAX：170kg
- 重量刻み：2.5kg

> 「保存」を押すと、未実施の今後メニューは新MAXで自動再計算されます。
> 過去ログは書き換わりません。今日のメニューを更新したい場合は「今日のメニューを再計算」ボタンを押してください（実施済みセットは保持）。

### トレーニング記録方法
1. ホーム画面の「今日のトレーニングを開始」を押す。
2. 種目ごとに各セットの重量・回数を入力し、達成チェックを付ける。
3. RPE（任意）と痛み（複数選択可）を選ぶ。
4. メモを書く。
5. 「セット完了+レスト」ボタンでセット完了 + レストタイマー自動開始。
6. 全種目を終えたら「トレーニング完了」を押す → 記録が保存される。

### レストタイマーの使い方
- 種目カード内の「レスト開始」または「セット完了+レスト」でタイマー起動。
- 種目ごとの初期レスト時間が自動セットされます（BIG3トップシングル=5分など）。
- `+30s` `-30s` `停止/再開` `リセット` `×` ボタンで手動操作可能。
- 終了時は画面が黄色く点滅 + ビープ音 + バイブ（対応端末のみ）でお知らせ。

### ブロック進行の考え方
- 1ブロック = 4ローテ × 8日 = 32日。
- 1ローテ目：通常負荷／2ローテ目：少し増量／3ローテ目：さらに増量／4ローテ目：疲労抜き（自動的にデロード負荷へ切替）。
- 各日のトレーニング完了時に「次のDayへ進みますか？」と聞かれます。
- **正式な次ブロック提案は4ローテ目Day8（休み）完了後に表示されます。** ブロック未完了の段階でブロック管理画面を開いた場合は「参考提案」として表示され、「提案を採用」「編集して採用」「次ブロックへ進む」ボタンはすべて無効です。
- 提案の判定ルール：
  - 全セット成功 / RPE8以下 / 痛みなし → 上げ提案
  - RPE9以上 → 据え置き
  - 失敗あり → 据え置き
  - 失敗多数 → 少し下げる
  - 痛みあり関連種目 → 据え置き

### 手動でDayを変更したい場合
- ホーム画面 →「Dayを手動変更」または「設定」画面から、ブロック・ローテ・Dayの数値を直接変更できます。

### 重量を手動調整したい場合
- 種目カードの「重量調整」ボタン。
  - 「今日だけ変更」：その日のセッションのみ反映
  - 「今後にも反映」：同じDay×種目×メニュー種別に対して差分を保存し、今後の予定重量に自動反映
- 設定画面下部の「手動調整一覧」で、いつでも解除できます。

## データ保存について

- 保存先：ブラウザの `localStorage`（キー：`mll_strength_planner_v1`）
- 保存内容：MAX設定、現在状態、ログ、手動調整、ブロック提案履歴、当日セッション
- 別ブラウザ・別端末・シークレットウィンドウ間ではデータは共有されません。
- データのバックアップ／復元には「エクスポート / インポート」（JSONファイル）をご利用ください。

### データリセット方法

- 設定画面下部「全データ削除」ボタン（2回確認あり）
- またはブラウザのデベロッパーツールから手動：
  ```javascript
  localStorage.removeItem('mll_strength_planner_v1');
  location.reload();
  ```
- 設定画面「初期値に戻す」はMAX設定のみ初期値に戻します（ログは保持）。

## GitHub Pagesでの公開方法

1. GitHub で空のリポジトリを作成（例：`myliftlog-strength`）。
2. 本フォルダ内のファイルをすべてリポジトリ直下に配置。
3. ローカルで初期化〜push：
   ```bash
   cd myliftlog-strength
   git init
   git add .
   git commit -m "initial commit"
   git branch -M main
   git remote add origin https://github.com/<your-username>/myliftlog-strength.git
   git push -u origin main
   ```
4. GitHubリポジトリの「Settings → Pages」で：
   - Source: `Deploy from a branch`
   - Branch: `main` / `(root)`
   - Save
5. 1〜2分待つと `https://<your-username>.github.io/myliftlog-strength/` で公開されます。
6. スマホでそのURLにアクセスし、Safari（iOS）／Chrome（Android）の「ホーム画面に追加」でPWAインストール完了。

> サブパスで公開するため、相対パス（`./manifest.json` など）で記述しています。リポジトリ名を変更しても動作します。

## 注意点

このアプリは医療やトレーナー指導の代替ではありません。痛みがある場合は無理に重量を上げず、必要に応じて休養してください。

## 既知の制限・簡易対応

- ログ画面のグラフは SVG での簡易折れ線（外部ライブラリなし）。詳細グラフ機能は未実装。
- 音声は WebAudio の簡易ビープのみ（音声ファイルなし）。
- ホーム画面「次のDay予定」は1日先のみ表示。
- RPE は単一値（1セットごとではなく種目全体）。
- service-worker のキャッシュはバージョン更新時に手動でブラウザ更新が必要な場合あり。
