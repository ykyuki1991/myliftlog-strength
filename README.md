# MyLiftLog Strength Planner

4メニューを順番に回しながら、メイン種目と補助種目を記録する個人用トレーニング管理PWAです。

1. 肩・腕（ミリタリープレス）
2. 脚（スクワット）
3. 胸（ベンチプレス）
4. 背中（ハーフデッド / 床引きデッドを実施回数で交互）

曜日には固定されません。休みや当日のメニュー変更を挟んでも、完了したメニューを基準に次へ進みます。旧8日ローテーションのログ、推定1RM履歴、MAX測定結果は履歴互換として引き続き閲覧できます。

## 主な機能

- メイン種目3セット×5回の記録と、種目別重量刻みによる次回候補
- 前回実績、経過日数、今回重量の理由表示
- 2回連続未達時の5%減候補（ユーザーが採用するまで自動反映しない）
- menuKey別の補助種目テンプレートと次回重量候補
- セット完了、RPE、痛み、メモ、レストタイマー
- 直近4回/8回の完遂率と部位別直接セット数
- 日別・月別ログ、メニュー/種目/役割フィルター
- 実測MAX、最新推定MAX、過去最高推定MAXの分離表示
- 期間指定の種目・部位休止
- JSONバックアップ/復元と旧データmigration
- オフライン起動対応PWA

RPE、痛み、メモは記録と確認のために保持します。これらを使って重量、休養、デロード、メニューを自動変更することはありません。

## データ保存

データはブラウザの`localStorage`へ保存されます。設定画面から定期的にJSONをエクスポートしてください。インポートは現在データを全置換します。

保存JSONには旧8日ログと4メニューログの両方が含まれます。読み込み時は不足している新設定だけを補完し、過去ログを一括書き換えません。

## 起動

静的なHTML/CSS/JavaScriptだけで動作します。PWAとして確認する場合はHTTPサーバー経由で開いてください。

```bash
python3 -m http.server 8080
```

## テスト

```bash
node --check app.js
node test_dom.js
node test_accessory.js
node test_progression.js
git diff --check
```

## ファイル

- `app.js`: データmigration、保存、メニュー生成、記録、分析、画面描画
- `styles.css`: スマホ優先UI
- `index.html`: 4タブ構成とPWA読込
- `service-worker.js`: オフラインキャッシュ
- `test_dom.js`: DOMとタイマー
- `test_accessory.js`: 補助種目
- `test_progression.js`: 進行、ログ、推定MAX、migration
