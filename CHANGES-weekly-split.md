# 週固定4分割への改修概要（feature/weekly-split-program）

8日周期ローテーション（block / rotation / day）を廃止し、週固定の部位別4分割へ移行しました。

## 週間スケジュール

| 曜日 | 部位 | メイン種目 |
|---|---|---|
| 日 | 肩・腕 | ショルダープレス 3×5（初週55kg） |
| 月 | 脚 | スクワット 3×5（初週135kg） |
| 火 | 休み | - |
| 水 | 胸 | ベンチプレス 3×5（初週95kg） |
| 木 | 背中 | ハーフデッド 3×5 162.5kg ⇔ 床引きデッド 3×5 145kg（週交互・自動管理） |
| 金・土 | 休み | - |

アプリを開いた日の曜日から当日メニューを自動表示します。

## 漸進・停滞ルール（自動提案）

- メイン: 規定セット×レップ完遂 → 翌週 +2.5kg。未達 → 同重量。
- 補助: 前週の使用重量を引き継ぎ。全セットで規定レップ到達 → 重量アップ提案（マシン系+5kg / ダンベル系+2kg）。
- 停滞（同重量で2週連続未達）→ 該当種目のみ約-10%で再スタート提案。固定ディロードなし。

## 変更ファイル

- `app.js` — 週次プログラムエンジン（スケジュール・デッド週交互・漸進/停滞提案・曜日別スロット）、
  週次セッション生成/保存、今日画面・計画画面・ヘッダーの週次表示、レストタイマー全種目300秒
- `index.html` / `service-worker.js` — アセットバージョン更新（PWAキャッシュ更新）
- `test_progression.js` / `test_accessory.js` — 週次エンジンのテスト追加、既存テストは互換モードで維持
- `CHANGES-weekly-split.md` — 本ファイル

## データ移行の挙動

- 既存ログ（8日ローテの記録）・推定1RM履歴・MAX測定結果は**削除も書き換えもしません**。
  ログ画面（日別/月別/MAX/推定MAX）から従来どおり閲覧できます。
- 新規記録は「日付＋曜日＋部位（weeklySplit/weekday/splitKey）」で保存され、
  block / rotation は使用しません（nullで保存）。
- 旧8日ローテのロジック（getDayMenu等）は表示互換のためコード上残していますが、UIからは使用しません。
- JSONエクスポート/インポートは store 全体を対象とするため、新旧データの両方を含めてそのまま動作します。
- 曜日別の補助スロットは初回はプリセット（依頼仕様の種目・セット×レップ・初期重量）で表示され、
  計画タブで編集すると `settings.weeklySlots` に保存されます。種目の追加・編集・削除・並べ替えが可能です。

## 初週の使い方

1. アプリを開くと当日の曜日メニューが表示されます（火・金・土は休み表示）。
2. メイン種目は初週重量（上表）が予定として入っています。セットを完了していくだけで記録されます。
3. 補助種目は初期重量ありのものは入力済み、無印のものは重量欄が空欄です。初回に重量を入力すると翌週から引き継がれます。
4. 「トレーニング完了」で保存すると、翌週の同曜日に漸進ルール適用後の重量が自動提案されます。
5. 背中の日のデッド種別（ハーフ/床引き）は初回利用の週を第1週として自動で交互になります。
6. レストタイマーはセット完了で自動起動（全種目5分）。±30秒ボタンで調整できます。
